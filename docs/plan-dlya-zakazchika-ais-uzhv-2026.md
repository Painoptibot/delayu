# Программа реализации АИС УЖВ (упрощённое ТЗ)

## Для представления заказчику — май — ноябрь 2026

**Документ-основание:** Техническое задание на оказание услуг по созданию АИС «УЖВ»  
(`ТЗ_АИС_УЖВ (2).docx`, согласовано УЖВ и УИКТиС)  
**Функциональный заказчик:** Управление по жилищным вопросам (УЖВ) — процессы, пользователи  
**Технический заказчик / закупка / приёмка:** Управление ИКТ (УИКТиС) — сервер, извещение, акт  
**Продукт в реестре:** **«ДелаЮ»** (Дела.ЮГИт), правообладатель **ЮГИт** — конфигурация **АИС УЖВ** (первая поставка)  
**Описание продукта:** `docs/delayu-product-opisanie.md`  
**Коммерция:** отдельный трек (Ателика, стройка, юрбюро) — см. `kommerciya-komplekt-2026.md`  

> **Важно:** в упрощённом ТЗ **внешние интеграции (ГИС ЖКХ, СМЭВ, Катарсис и др.) не входят в объём первого этапа** — опции I-01…I-14 **не активны**, учёт межведомственных ответов **вручную**, обмен файлами **CSV/XLSX** и скриптами **PY-01…PY-07**.  
> **Срок по ТЗ:** оказание услуг **до 02.10.2026** (п. «Условия контракта»). План ниже увязывает **приёмку к концу сентября — началу октября** с этой датой; перенос на ноябрь возможен только **изменением ТЗ/контракта**.

## 1. Чем упрощённое ТЗ отличается от полной версии 1.1


| Аспект                            | ТЗ v1.1 (рабочий стол)        | **Согласованное ТЗ (2)**                               |
| --------------------------------- | ----------------------------- | ------------------------------------------------------ |
| Интеграции                        | ГИС ЖКХ, СМЭВ, Катарсис и др. | **Нет** — «по умолчанию ни одна опция I-xx не активна» |
| Межведомственные запросы          | Авто через СМЭВ               | **Ручной учёт** + отчёт ОТЧ-8                          |
| Обмен данными                     | API внешних ИС                | **Импорт/экспорт файлов**, скрипты Python              |
| Срок контракта                    | В договоре                    | **До 02.10.2026**                                      |
| Модули учёта, договоров, контроля | Да                            | **Да** (функционал сохранён)                           |


**Для заказчика это плюс:** реалистичнее уложиться в календарь май–октябрь, меньше зависимость от смежных ведомств и техвозможностей МФЦ/Госуслуг.

## 4. Май — июнь: прототип и очные сессии

### 4.1. Состав прототипа (к концу июня)

**Платформа:** веб, JWT, роли, журнал аудита, PostgreSQL, размещение в контуре заказчика (по ТЗ — Astra Linux, Nginx, Docker).

**Подсистемы (по разделу «Требования к функциям» упрощённого ТЗ):**


| Код  | Подсистема          | В прототипе (июнь)                                  | К приёмке 02.10             |
| ---- | ------------------- | --------------------------------------------------- | --------------------------- |
| —    | Малоимущие + расчёт | Заявление, расчёт, заключение                       | Полный цикл                 |
| —    | Учёт нуждающихся    | Реестры, очерёдность                                | + история, отчёты           |
| —    | Молодые семьи       | Списки (базово)                                     | По законам КК 2704, 2710    |
| —    | Дети-сироты         | Карточка дела, **ручная** фиксация решений Минтруда | Без I-04, I-08              |
| —    | Договоры            | Реестр договоров                                    | + шаблоны docx/pdf          |
| —    | Жилфонд             | МКД, помещения                                      | Импорт PY-01                |
| —    | Жилконтроль         | Проверки, предписания                               | Реестр проверок             |
| —    | Переселение         | Реестры аварийного фонда                            | Ручной ввод / файлы         |
| —    | Обращения           | Регистрация, сроки 30 дн.                           | + ответы, ОТЧ-5             |
| —    | Отчёты              | ОТЧ-1, ОТЧ-5                                        | ОТЧ-1…ОТЧ-10 (где без I-xx) |
| —    | Шаблонизатор        | 1–2 образца                                         | docx/pdf по ТЗ              |
| I-xx | Интеграции          | **Не делаем**                                       | Заглушки/справочник опций   |


**Обмен с внешним миром на этом этапе:** только **файлы** и **PY-скрипты** (импорт справочников, миграция, отчёты, ZIP по делу).

### 4.3. Блоки вопросов для анкетирования (14+)

1. Актуальные регламенты и приказы (в т.ч. пр. № 58/2026).
2. Образцы заявлений, заключений, постановлений.
3. Алгоритм расчёта дохода и имущества (краевые правила).
4. Категории учёта и очерёдность.
5. Формы отчётности ОТЧ-1…ОТЧ-10 — какие обязательны к 02.10.
6. Роли и матрица утверждений.
7. Регламент обращений (30 дней).
8. Ведение дел детей-сирот **без СМЭВ** — как фиксируют ответы ведомств сейчас.
9. Жилфонд: кто ведёт данные (УЖВ / Горжилхоз), формат выгрузок.
10. Жилищный контроль — перечень предметов проверок (таблица ТЗ).
11. Контур размещения (сервер администрации, VPN).
12. Требования ИБ и 152-ФЗ.
13. Способ закупки и НМЦК (вводные для августа).
14. Обезличенные данные для пилота / миграции (PY-04).

## 5. Июль: реестр отечественного ПО + разработка

**Реестр:** подача в начале июля → **запись до конца июля / начала августа** (с учётом замечаний).

**Параллельная разработка (без интеграций):**

- Все подсистемы из п. 3.1 — рабочие сценарии в продуктивной ветке.  
- PY-01, PY-03, PY-04 — импорт и миграция.  
- PY-02, PY-06 — отчёты и пакеты документов.  
- Шаблонизатор docx/pdf.  
- CI/CD, Docker, документация (черновики по разделу «Требования к документированию»).

**Не делаем в июле:** настройка I-01…I-14, СКЗИ на каналах к внешним ИС (только по требованию заказчика для VPN/HTTPS).

## 6. Август: реестровый номер и закупка

1. Получение **реестровой записи** (номер для извещения).
2. Дополнение описания объекта закупки:
  - функции по согласованному прототипу;  
  - ПО **в реестре** на дату заявки;  
  - **отсутствие** обязательных интеграций I-xx в базовом объёме;  
  - приложение: протоколы визитов 1–4.
3. **Публикация закупки** (начало августа + срок подачи заявок).
4. Обоснование НМЦК (разработка + внедрение + документация + 12 мес. сопровождения — если заложено в контракте).
