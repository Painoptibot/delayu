# АИС УЖВ — документация разработки

## Архитектура «платформа ↔ подсистема»

| Уровень | Кто | Что настраивает |
|--------|-----|-----------------|
| **Платформа «ДелаЮ»** | `admin` (superuser, ЮГИт) | Каталог подсистем, глобальные интеграции (M42–M45), почтовый транспорт, инфраструктура, лицензии |
| **АИС УЖВ** | `uzhv_admin` | Пользователи, роли, НСИ, формы, BPM, шаблоны уведомлений в контуре УЖВ |
| **Пользователь УЖВ** | `uzhv_spec`, `uzhv_mgr` | Рабочие модули: учёт нуждающихся, жилфонд, договоры, обращения |

Глобальные настройки недоступны администратору подсистемы (фильтр меню + middleware по URL).

## Быстрый старт

```bat
.venv\Scripts\python.exe manage.py migrate
.venv\Scripts\python.exe manage.py seed_uzhv
runserver.bat
```

Вход:
- **Платформа:** `admin` / `admin` → переключить контур на «Пилот» или управлять подсистемой `uzhv` из каталога
- **УЖВ:** `uzhv_admin` / `uzhv_admin` → меню «АИС УЖВ»

## Скрипты PY-01…PY-07 (CLI)

| Код | Команда | Назначение |
|-----|---------|------------|
| PY-01 | `manage.py uzhv_py01_import <файл>` | CSV жилфонд / XLSX договоры |
| PY-02 | `manage.py uzhv_py02_reports` | Пакетный экспорт ОТЧ в каталог |
| PY-03 | `manage.py uzhv_py03_validate <csv>` | Проверка СНИЛС/паспорта |
| PY-04 | `manage.py uzhv_py04_migrate <csv> mapping.json` | Миграция legacy |
| PY-05 | `manage.py uzhv_py05_recalc_queue` | Пересчёт очереди |
| PY-06 | `manage.py uzhv_py06_case_zip <id>` | ZIP по учётному делу |
| PY-07 | `manage.py uzhv_py07_notify_payload` | JSON для уведомлений |

Пример mapping: [migration/mapping_case.example.json](./migration/mapping_case.example.json)

После импорта дел или изменения категорий: `manage.py uzhv_py05_recalc_queue --subsystem uzhv`

Подробнее: [rukovodstvo-polzovatelya.md](./rukovodstvo-polzovatelya.md) §8.
