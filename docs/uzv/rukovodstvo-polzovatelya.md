# Руководство пользователя АИС «УЖВ» (текущая версия)

**Продукт:** платформа «ДелаЮ», конфигурация АИС УЖВ  
**Подсистема:** `uzhv`  
**Основание:** упрощённое ТЗ `ТЗ_АИС_УЖВ (2).docx` (без интеграций I-01…I-14 на этапе 1)

> Полная приёмка по ТЗ **ещё не завершена** — см. [статус реализации](#статус-относительно-тз). Ниже описана **фактическая** работа с уже внедрёнными модулями.

---

## 1. Запуск и вход

```bat
.venv\Scripts\python.exe manage.py migrate
.venv\Scripts\python.exe manage.py seed_uzhv
runserver.bat
```

Откройте в браузере: `http://127.0.0.1:8000/`

| Логин | Пароль | Роль |
|-------|--------|------|
| `uzhv_admin` | `uzhv_admin` | Администратор контура УЖВ |
| `uzhv_mgr` | `uzhv_mgr` | Начальник УЖВ |
| `uzhv_spec` | `uzhv_spec` | Специалист по учёту |
| `uzhv_orphan` | `uzhv_orphan` | Специалист (дети-сироты) |
| `uzhv_contract` | `uzhv_contract` | Специалист по договорам |

После входа откройте **главную УЖВ**: `/uzhv/` или выберите подсистему «АИС УЖВ» в переключателе (если у пользователя несколько контуров).

**Платформенный администратор** (`admin` / `admin`) управляет каталогом подсистем и глобальной инфраструктурой; настройки УЖВ — через `uzhv_admin`.

## 3. Типовые сценарии

### 3.1. Постановка на учёт (учётные дела)

1. **Граждане** → «Создать» — ФИО, контакты, СНИЛС (при наличии).
2. **Учётные дела** → «Создать» — выберите гражданина, категорию учёта, дату постановки.
3. **Очерёдность** (queue_position) пересчитывается автоматически (PY-05): льготная категория → дата учёта. Кнопка «Пересчитать очередь» в реестре дел.
4. В карточке дела — вкладка **«Малоимущие»** (`/low-income/`): регистрация даты заявления (срок **30 дней** из НСИ), состав семьи, доход, имущество, **сканы вложений**, расчёт признака малоимущего по прожиточному минимуму (НСИ `uzhv_subsistence_minimum`). Просрочки — в **Календаре сроков**.
5. Реестр дел: фильтры **муниципальная подведомственность** / **отдельная категория** (сироты, льготники).

### 3.2. Молодые семьи и дети-сироты

- **Молодые семьи:** реестр `/uzhv/young-families/` — программа, критерии, статус участия.
- **Дети-сироты:** реестр `/uzhv/orphans/` — статус реализации права, решение Минтруда вводится **вручную** (интеграция I-04 не активна). **Пакет документов (ZIP):** кнопка в карточке или `/uzhv/cases/<id>/package.zip`.

### 3.3. Жилфонд и расселение

1. **Жилфонд** → создайте **МКД** (адрес, состояние, признак аварийности).
2. В карточке МКД — **помещения** (площадь, этаж, статус: свободно / занято / аварийно).
3. **Расселение** — список домов в программе переселения (`in_resettlement_program`).

### 3.4. Договоры

- Просмотр реестра: `/uzhv/contracts/`.
- **Создание / редактирование:** `/uzhv/contracts/new/`, `/uzhv/contracts/<id>/edit/`.
- **Расторжение:** снять «Действует», указать основание — помещение освобождается.
- **Импорт из Горжилхоз:** `/uzhv/contracts/import/` — xlsx по шаблону из `docs/uzv/form/`.

### 3.4a. Малоимущие и печатные формы

- `/uzhv/cases/<id>/low-income/` — состав семьи, расчёт, скачивание **docx** (заключение, справка).
- Паспортные данные — `/uzhv/citizens/<id>/edit/`.

### 3.5. Обращения граждан

1. **Обращения** → «Создать» — заявитель, учётное дело (опционально), тема, исполнитель.
2. Автоматически: входящая корреспонденция M24, **срок ответа 30 дней**, запись в истории статусов.
3. Контроль — **Календарь сроков** /uzhv/deadlines/ и сводка просрочек на /uzhv/.
4. Редактирование /uzhv/appeals/<id>/edit/: вложения, **вид заключения**, текст ответа.
5. При статусе «Ответ дан» / «Закрыто» — регистрация **исходящего ответа** в M24 (связь с входящим).
6. Проект ответа — /uzhv/appeals/<id>/response.docx.

### 3.6. Жилищный контроль

Цепочка: **План проверок** → **Предписание на проверку** → **Проверка** → **Предписание об устранении** → **Протокол АП** → **Судебное дело** / **Исп. производство**.

1. /uzhv/inspection-plans/ — план внеплановых проверок (период, основание).
2. /uzhv/inspection-orders/new/ — предписание на проведение проверки (адресат, срок проведения, связь с планом).
3. Из карточки предписания — «Создать проверку» (автозаполнение объекта и предмета).
4. /uzhv/inspections/new/ — тип, объект, даты, результат.
5. При нарушениях — предписание об устранении с датой исполнения.
6. Протокол об административном правонарушении — `/uzhv/admin-protocols/`.
7. Судебное дело — /uzhv/court-cases/; исполнительное производство — /uzhv/enforcement/.

### 3.7. Межведомственные запросы (ручной режим)

`/uzhv/interagency/` — регистрация запроса (тип: Соцзащита, Росреестр, БТИ и др.), канал **MANUAL** или **FILE**, фиксация ответа и даты. Отчёт **ОТЧ-8** формируется из этих данных.

### 3.8. Отчётность

Хаб: `/uzhv/reports/`

| Код | Содержание | Экспорт |
|-----|------------|---------|
| ОТЧ-1 | Список на учёте | CSV / XLSX / PDF |
| ОТЧ-2 | Предоставление жилья за период | + параметры `?from=&to=` |
| ОТЧ-3 | Договоры соцнайма (Горжилхоз) | |
| ОТЧ-4 | Результаты проверок | + период; колонки: протоколы АП, № предписания на проверку, план |
| ОТЧ-5 | Статистика обращений | + период |
| ОТЧ-6 | Движение жилфонда (поступило/выбыло за период) | + период, XLSX с шапкой |
| ОТЧ-7 | Дети-сироты | |
| ОТЧ-8 | Межвед. запросы | + период |
| ОТЧ-9 | Расселение аварийного фонда (4779) | XLSX со сводкой |
| ОТЧ-10 | Исполнение предписаний и судов | + период; блоки: предписание на проверку, об устранении, исп. производство |

Пример: `/uzhv/reports/otch-1/export.xlsx`

### 3.9. Массовые операции и экспорт

- `/uzhv/bulk/` — смена статусов дел, обращений, предписаний; закрытие договоров.
- Экспорт CSV из списков и `/uzhv/deadlines/export.csv`, `/uzhv/overdue/export.csv`.

### 3.10. Мобильные уведомления (PWA)

- В **Личном кабинете** → «Push УЖВ» — подписка на уведомления о просрочках.
- Service worker: `/uzhv/sw.js`.

## 8. Скрипты PY-01…PY-07 (cron / сервер)

Все команды выполняются из корня проекта. Код выхода: `0` — успех, `1` — ошибки валидации, `3` — конфигурация.

```bat
manage.py uzhv_py01_import data\fund.csv --subsystem uzhv --kind fund
manage.py uzhv_py01_import data\contracts.xlsx --kind contracts
manage.py uzhv_py02_reports --output media\uzhv_exports --format both
manage.py uzhv_py03_validate data\citizens.csv --report media\validate_errors.csv
manage.py uzhv_py04_migrate data\legacy.csv docs\uzv\migration\mapping_case.example.json
manage.py uzhv_py05_recalc_queue --dry-run
manage.py uzhv_py05_recalc_queue
manage.py uzhv_py06_case_zip 12 --output media\packages\delo.zip
manage.py uzhv_py07_notify_payload --days-ahead 5
```

**PY-05 (очередь):** льготная категория → дата постановки на учёт. Дела «дети-сироты» и «льготная» получают более высокий приоритет, чем «общая очередь».

**Рекомендуемый cron (Linux):**

- `0 2 * * *` — `uzhv_py05_recalc_queue`
- `0 6 * * *` — `uzhv_py02_reports` + `uzhv_py07_notify_payload`
- `0 3 * * 0` — `uzhv_py02_reports --codes otch-1,otch-3`

## 9. Статус относительно ТЗ

| Область | Готовность | Комментарий |
|---------|------------|-------------|
| Реестры (дела, граждане, фонд, договоры) | ~88% | История статусов, согласия по договорам |
| Малоимущие | ~88% | СНИЛС/паспорт в составе семьи, назначение исполнителя, отказ, docx |
| Молодые семьи / сироты | ~85% | Критерии, docx, спец. помещения, проект постановления |
| Жилфонд / расселение | ~90% | ЛС, выписки, частный фонд, непригодные |
| Жилконтроль | ~90% | Планы, предписания на проверку и об устранении, УФССП |
| Обращения + SLA | ~88% | M24 in/out, история, вложения, docx |
| Межвед. (ручной) | ~85% | Реестр + ОТЧ-8 |
| Отчёты ОТЧ-1…10 | ~92% | ОТЧ-4/10 расширены под новые реестры |
| PY-01…PY-07 (CLI) | **~85%** | management-команды `uzhv_py01`…`uzhv_py07` |
| Автоочередь | **~88%** | PY-05 + авто после дел/расчёта |
| Шаблоны docx/pdf УЖВ | ~78% | + ответ на обращение |
| Интеграции I-xx | N/A | Вне объёма этапа 1 |

**Итог:** готовность in-scope **~92–93%**. До приёмки: сверка верстки XLSX/docx с образцами заказчика на визите.
